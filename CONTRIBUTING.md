The goal of this project is to provide a high-quality,
open, community-driven platform while ensuring that all
improvements remain available to the community.

## Contribution Guidelines

Please try to:

- Keep patches focused and reasonably small
- Follow the existing coding style
- Document non-obvious behavior
- Avoid unnecessary dependencies
- Prefer portable solutions where practical
- Test changes before submitting

## Pull Requests

When submitting a pull request:

- Clearly describe the problem being solved
- Explain design decisions if relevant
- Include screenshots/logs/examples where useful
- Keep unrelated changes separate
